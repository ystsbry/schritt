# implement-plan

あなたは経験豊富なソフトウェアエンジニアです。与えられた**実装計画の全ステップ**を、先頭から順に対象リポジトリへ実装し、ステップごとにレポートを出力します。1ステップずつ確実に実装する流儀は implement-step と同じですが、こちらは**計画全体を一括**で進めます（`schritt implement` コマンド相当）。

## 入力

```
/schritt:implement-plan <REFINEMENT_DIR> [--repo <REPO_DIR>]...   # Claude Code (plugin)
$implement-plan <REFINEMENT_DIR> [--repo <REPO_DIR>]...           # OpenAI Codex CLI
```

- `<REFINEMENT_DIR>` (任意): リファインメント結果ディレクトリ（`~/.schritt/pbi-{番号}/{日時}/`）のパス。
  - `<REFINEMENT_DIR>/implementation/NN-*.md` … 実装ステップ群（`01-`, `02-` の連番順に実装する）
  - `<REFINEMENT_DIR>/pbi.md` … 元のPBI（文脈把握のため。存在すれば読む）
  - `<REFINEMENT_DIR>/notes.md` … 補足メモ（存在すれば読む）
- `--repo <REPO_DIR>` (任意・**複数指定可**): 実装対象のリポジトリ。ここに**コードを実装する**。

`<REFINEMENT_DIR>` が渡されず、チャットで直接ステップ群を貼り付けられた場合は、その内容を対象として扱う。

## 手順

1. `implementation/` 配下の `*.md` を**ファイル名順（01-, 02-, …）に列挙**する。`pbi.md` / `notes.md` で全体の文脈を把握する。
2. 対象リポジトリの既存コードとテストの書き方（ディレクトリ構成・命名・テストフレームワーク）を調査する。
3. 各ステップを**先頭から順番に**実装する。**前のステップの実装結果を踏まえて**次のステップに進むこと（順序を守る）。各ステップで:
   - そのステップの範囲を実装する
   - 対応する**単体テスト**を書く
   - 可能であればテスト/ビルドを実行し、通ることを確認する
   - そのステップのレポートを書き出す（下記「出力」）
4. 全ステップ完了後、何を実装したか・残課題の有無を**全体サマリ**として簡潔に述べる。

## 出力

### `<REFINEMENT_DIR>` が渡された場合

各ステップのレポートを、**そのステップと同じファイル名**で `<REFINEMENT_DIR>/reports/` 配下に書き出す（例: `implementation/02-api.md` → `reports/02-api.md`）。各レポートは最低限、次の見出しを含める。

```
# <ステップ名>

## 実装した内容
- 変更/追加したファイル、追加した関数・型、設計上の判断

## 書いた単体テスト
- 追加したテストファイルとテストケース
- テストを実行できた場合はその結果（pass/fail）

## 補足
- 未対応・TODO・POへ確認したい点など（あれば。無ければ「特になし」）
```

リポジトリへの実装（コード変更）と、各 `reports/<ファイル名>` の生成の**両方**を必ず行うこと。この出力レイアウトは `schritt implement` コマンドと同一なので、生成後は `schritt view` でそのまま閲覧できる。

### チャットで直接依頼された場合

各ステップのレポートを、**ステップごとに見出しを分けた1つのマークダウン**として返信に出力する。
