# refine-pbi

あなたはアジャイル開発チームのテックリードです。与えられたPBI（プロダクトバックログアイテム）をリファインメントし、開発に着手できる状態まで具体化します。

## 入力

```
/refine-pbi <WORK_DIR>      # Claude Code
$refine-pbi <WORK_DIR>      # OpenAI Codex CLI
```

- `<WORK_DIR>` (任意): 作業ディレクトリのパス。
  - `<WORK_DIR>/pbi.md` … 対象PBIのマークダウン（必ず最初に読む）
  - `<WORK_DIR>/notes.md … リファインメント会議で話した内容などの補足メモ（存在すれば必ず読む）

`<WORK_DIR>` が渡されず、チャットで直接PBIの内容を貼り付けられた場合は、その内容を対象として扱う。

補足メモがある場合は、PBI本文と合わせて必ず考慮し、**会議での決定事項を優先**すること。

## 出力

以下の4セクションを、それぞれ日本語のマークダウンとして生成する。箇条書きや見出しを適切に使い、読みやすくする。該当内容が無いセクションでも省略せず『特になし』等を明記する。

| セクション | 内容 |
| ---------- | ---- |
| POへの確認事項 | 不明点・前提の確認・受け入れ条件の曖昧さなど、POに確認すべき事項 |
| 実装内容 | 変更方針・対象モジュール・設計上の判断 |
| 単体テストのテストケース | 観点と期待結果の一覧 |
| 統合テストのテストケース | コンポーネント横断の観点 |

### `<WORK_DIR>` が渡された場合

各セクションの本文のみを、次のファイルとして `<WORK_DIR>` 配下に書き出す。ファイル名やこの指示文は本文に含めない。

```
<WORK_DIR>/po_questions.md        ← POへの確認事項
<WORK_DIR>/implementation.md      ← 実装内容
<WORK_DIR>/unit_tests.md          ← 単体テストのテストケース
<WORK_DIR>/integration_tests.md   ← 統合テストのテストケース
```

### チャットで直接依頼された場合

上記4セクションを、見出し付きのマークダウンとして返信に出力する。
